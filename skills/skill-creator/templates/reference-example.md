## Reference Document Guidelines

**Structure:**
1. Start with overview (what and why)
2. Core concepts (fundamentals)
3. Detailed documentation (how)
4. API reference (if applicable)
5. Patterns and practices
6. Troubleshooting
7. Advanced topics
8. Complete examples

**Best practices:**
- Use clear headings and subheadings
- Include plenty of code examples
- Explain both what and why
- Cover common mistakes and solutions
- Link to related topics
- Keep examples concrete and runnable
- Use consistent formatting
- Add comments to complex code

**When to create reference documents:**
- Topic is too detailed for SKILL.md
- Need comprehensive API documentation
- Multiple related concepts to explain
- Troubleshooting guide is needed
- Advanced usage requires deep explanation

**When to split into multiple references:**
- Document exceeds 1000 lines
- Multiple distinct topics covered
- Different audiences (beginner vs advanced)
- Logical separation of concerns
